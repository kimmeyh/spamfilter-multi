================================================================================
                    ARCHITECTURE HIGHLIGHTS
================================================================================

Database-First Design:
  SQLite (Primary) → RuleSetProvider → YAML (Secondary)

  Benefits:
  - Scalability: Supports thousands of patterns
  - Consistency: Single source of truth
  - Versioning: YAML enables git history
  - Flexibility: JSON structure for future enhancements

Smart Pattern Matching:
  Email:   user@example.com → ^user@example\.com$
  Domain:  @example.com     → ^[^@\s]+@example\.com$
  Regex:   Full regex       → Used as-is

Exception Evaluation:
  1. Normalize email
  2. Check safe sender pattern match
  3. If matched, check exceptions
  4. Return true only if no exception matches

Interface-Based Design:
  - RuleDatabaseProvider interface for testability
  - MockRuleDatabaseProvider enables fast iteration
  - Enables future database implementations
